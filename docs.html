Getting Started

This part of the doc will help you to quick start your project and will give you a basic idea about how Nova works.

How to include Nova to your project

In order to fully use nova components you need to include the following files to your project:

  • 1. nova.css: this file can be found under nova/css/nova.css

  • 2. animate.css: this file can be found under vendor/css/animate.css

  • 3. pe-icon-7-stroke.css: this file is needed in order to use the Stroke 7 icons library

  • 4. A reference to the Open Sans font

  • 5. jquery-3.3.1.min.js: this file can be found under vendor/js/jquery-3.3.1.min.js

  • 6. nova.js: this file can be found under nova/js/nova.js

  • 7. Copy the nova/img folder and paste it so it's at the same level as your css and js folder


That's the minimum required setup to used most of the nova components, If you wish to customize your nova.css please refer to the SASS section in the docs.

Some nova components such as Tables, dropzones, selects require some additional javascript. In the complex components section of the docs you can find more information about each component and which javascript is necesary in order to make it work.

SASS

Nova is built using SASS. This allows you to customize you nova.css file, meaning you can include only the components (modules) you are going to use and ommit the rest. In this section we will explain the folder structure of the SASS modules and also how to build your own customized version of nova.

BEM

Nova uses BEM as a name metodology. BEM stands fot Block Element Modifier.

Each element is named with the __ syntax

Each modifier is named with the --syntax

SASS Folder Structure



abstract folder: in this folder you will find the _mixins.scss file and the _variables.scss file.

  • _mixins.scss :

    this file contains all the mixins used troughout the sass project. If you need to add some of your own this is the place to do so.

  • _variables.scss :

    this file contains mainly the variables related to the nova color pallete. If you want to change a particular color you should do it here and that change will propagate to the entire proejct.

base folder: this folder contains the _base.scss, _typography, and _utilities.scss files.

  • _base.scss::

    the css reset can be found here, also styles applied to the html and body tag

  • _typography.scss::

    this file contains all the styles for the html elements related to typography: headers, paragraphs, spans, labels etc.

  • _utilities.scass:

    is where utilities-namespaced classes are written, like u-text-center, u-bold, u-mt-small

components folder

This folder contains all the heavy lifting logic.

It contains the styles related to every single nova component. If you need to customize any particular component you can do it here, for example you can change a particular style of the alert in the file _alert.scss.

dashboardComponents folder

This folder contains all the heavy lifting logic related to the dashboard components

All the styles associated with the dashboard components can be found here. The dashboard components are displayed in the index.html file.

pages folder

This folder has styles related particulary to some pages of the dashboard. Here you should write any style that applies only to a particular page in your project.

main.scss file

This is the file that includes all the other files discussed above.

Here is where you can play around and only include those files you will end up using. For example if you know that your proejct won't be using a modal, then there is no need to include the file _modal.scss

By doing this you can assure you only include those components you will end up using. This will result in an even more light weighted nova.css file.

Building your customized nova.css file

Once you downloded the project make sure to run npm install to install all the npm dependencies.

To build the final nova.css file you should run npm run build:css. This will compile the sass files into css, concat, prefix and compress the final nova.css file.

You can find the build nova.css file under nova/css/nova.css

Complex Components

Notifications

Nova has it's own notification's api. You just need to pass the container you want the notification to be attached to and then some options to configure it.

You can play around in the notification builder page to try out different examples.

Example:

Notification's API:

options object properties description

message

:String this is the message to display to the user.

Example: "You have 10 new messages"

style

:String The background color is determined by this property and this are the possible values: success, danger, warning, info

position

:String The position that the final nofitication will have, this are the possible values: top-right, top-left, bottom-left, bottom-right

type

:String There are three notification types: simple, bar, image.

timeout

:Number The lifespan of the notification in ms. If a value is not providad then the notification will exist until user closes it.

Example: 3000

animationIn

:String The incoming animation the notification will have. You can choose from a wide variaty of animations. Please refer to the following link to see a full list of the available animations: animate.css

Example: lightSpeedIn

animationOut

:String The outgoing animation the notification will have. You can choose from a wide variaty of animations. Please refer to the following link to see a full list of the available animations: animate.css

Example: lightSpeedOut
